Tirando leite de pedra com o plugin Autoptimize no WordPress + Plugins de Cache

Famoso na galeria do WordPress, o plugin Autoptimize tem mais de 900.000 downloads. Mas qual a finalidade dele?

Quando comecei a desenvolver sites com o wordpress, eu nem imaginava que esse tipo de plugin existia. Podemos considerar o autoptimize sendo da categoria de plugin otimizadores.

1. Dando uma Pincelada

É importante saber o que o plugin faz. Isso vai te ajudar na hora da escolha e a resolver possíveis pepinos.


Autoptimize torna bem fácil otimizar seu site. Pode agregar, minificar e armazenar (cache) scripts e estilos, adicionar CSS no elemento head da página por padrão (mas também pode adiar), move e adia scripts para o rodapé e minifica HTML.

Criador: Frank Goossens (futtta)

“Vixi, não entendi nada”. Se liga ai!

1.1 Minificar

Minificar: O processo de minificação nada mais é do que retirar os espaços em branco do código. Isso Faz com que ele seja menor e consequentemente reduz o tempo de carregamento do site.

Agora imagine milhares de linhas de código no seu site. Minificar agora parece ser uma boa ideia.

1.2 Agregar (Concatenar)

Agregar tem como objetivo combinar vários arquivos em um único só. Isso resulta em menos solicitações HTTP. Na prática, faz com que o navegador vá buscar menos vezes a informação no servidor. E menos solicitações HTTP é extremamente importante para a hospedagem compartilhada, que é usada em mais 92% das hospedagens WordPress.

O Autoptimize ainda faz mais. Ele comprime todos os scripts e CSS , adiciona cabeçalhos de expiração, e armazena tudo em cache.

2. Análise de velocidade (antes)

Antes de configurar o plugin precisamos fazer uma análise da velocidade
de carregamento e de outros parâmetros do site para poder comparar o antes e depois. Se não sabe como fazer, eu explico com mais detalhes nesse tutorial

Eu criei um ambiente de testes online V1.0 . Todos os parâmetros serão baseados por ele. Se quiser saber quais configurações e plugins estão sendo usados, clique aqui

Fully Loaded Time – Tempo total de carregamento da página (segundos).

Total Page Size – Tamanho total da página (Megabytes).

Requests – Esse é zica. Solicitações ao servidor. (Quanto mais, muito pior).

3. Principais configurações. Aba (JS, CSS, HTML)

Para acessar, vá em: Painel/Configurações/Autoptimize

Otimizar Código CSS – Ao ativar irá agregar e minificar todo o CSS do seu site e irá agregar o CSS no HTML

Otimizar Código Javascript – Ao ativar irá agregar e “linkar” o Javascript do seu site

Otimizar Código HTML – Ao ativar irá minificar o HTML do seu site

URL base da CDN – Caso você use um CDN, pode inserir a URL aqui. Como eu uso e indico o CDN da Cloudflare, essa opção não se aplica aqui. Deixe em branco

Ative as 3 opções e clique no botão SALVAR MUDANÇAS

4. Configurações Avançadas. Aba (EXTRA)

Fontes Google – Hoje, a maioria dos sites usam fontes do Google. Elas são bonitas e há dezenas de fontes para escolher. Mas atrasam a carregamento do site. Ao ativar essa opção, as fontes do Google irão carregar mais rapidamente.

Remove emojis – Nunca ví um site WordPress com essas carinhas. Os emojis atrasam o carregamento do site. Ao ativar, eles serão removidos.

Usar “Preconnect” para domínios de terceiros (usuários avançados) – Ajuda os navegadores a renderizarem com antecedência algumas solicitações externas. O navegador precisa buscar a informação em outro servidor e depois exibir no seu site.

5. Análise de velocidade (depois)

Fully Loaded Time – Subiu. 100ms. Aceitável.

Total Page Size – Caiu. Bom. O plugin fez o trabalho.

Requests – O mais importante. Aqui o plugin mostra sua real utilidade. Eu Curti. de 46 para 32 solicitações. A pontuação também subiu.

Agora vamos juntar o Autoptimize com um plugin de cache.

6. Plugin de Cache

O Autoptimize trabalha muito bem ao lado de um plugin de cache. Neste tutorial vamos configurar o plugin WP Super Cache, que é um plugin oficial dos desenvolvedores do WordPress. Há vários plugins de cache bons. Você pode acessar o tutorial que eu desenvolvi para o o WP Super Cache, como também para o WP Rocket e o W3 Total Cache.

Foi um grande resultado, apenas usando 2 plugins. Há muito o que melhorar ainda.
